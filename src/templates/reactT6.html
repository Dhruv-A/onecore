6. States

6.1 Difference between a prop and state

States are very similar to props except they have one important difference. Props are passed to a component whereas states are managed within components.

6.2 Rules of states

  • Only useable with class based components (or with functional components using hooks component)
  • A state is a JS object that contains some amount of data that is relevant to a singular component. i.e. Only contains some amount of information relevant to what we want
  • Updating state or properties will cause the component to almost instantly rerender
  • States must be initialised when a component is first created
  • States can only be updated using the function ‘setState’
