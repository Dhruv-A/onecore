3. Functional Components

3.1 Main parts of a component

It is important to understand the 3 main parts of a component:

  • Component Nesting
  • Component Reusability
  • Component Configuration

I'll be covering these over several modules.

3.2 Component Nesting

Let's start by editing the app we created in the introduction

Step 1: You’ll likely see many files under the src folder of your newly generated application. Surprisingly enough, these are all pre-generated and you can actually delete all of them
Step 2: Create an empty file called index.js (This will serves as the root file for your entire application)

Step 3: Now inside the file, we’ll start by importing two PACKAGES (These are essential for rendering our components onto).
Importing react allows the Javascript file to interpret JSX, you are telling the code compiler, “Hey! I’m providing you JSX in the file”
ReactDOM allows us to render elements onto the dom with ReactDOM.render
NOTE: It is case sensitive - make sure your imports are defined as follows in index.js:

import React from 'react';
import ReactDOM from 'react-dom';

Step 4: Now that you’ve got your imports create a new file called App.js (in the src folder) and try importing React. Also note that, this time we do not need to import ReactDOM as the rendering stuff is done in the root index.js file.

import React from 'react';

const App = () => {
  return (
    <div>Hello World</div>
  );
};

export default App;

3.3 Explaining the previous code

Alright now breaking down this chunk of code. Here I’ve created a functional component, making use of the ES6 Arrow function to define a component called App and returning the text Hello World. By convention, it is common to name the component the name you have named the file e.g [Name of app].js 

const [Name of App] = () => {

Whatever element you return from the component should be wrapped in div tags. Finally to make this component accessible to the rest of the app we need to export it with export default [name of App].

Step 5: Cycling back to the root index.js file

import React from 'react';
import ReactDOM from 'react-dom';

//Import the App file
import App from './App';

//Rendering out the App component
ReactDOM.render(
  <App />,
  //rendering it out on the div with root id in index.html [public folder]
  document.querySelector('#root')
)

In the previous step we exported the App functional component we created. Now it only makes sense to import it to our root file and render the App component into the DOM -> This will allow us to see it on our browser. Note: import App from ‘./App’; the ‘./’ syntax means from the current directory (See how App.js and index.js are in the same folder?).

Making use of the ReactDOM library to manipulate the dom, we use ReactDOM.render() to render out the app component.

  • Imported components are represented in the syntax <[component] /> and finally we use document.querySelector(‘#root’) to tell the compiler where to render it. If you look in the public folder, index.html you will see a div with the id root. This is essentially where we render the component.
  • Now to see it all in your browser, head over to command prompt or Terminal if you’re using a mac and type npm start.

Congratulations! You just made a functional component.
