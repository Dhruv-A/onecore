2. Fundamental Theory

2.1 React vs ReactDOM

Before getting into React and ReactDOM, you have to understand the problem with simple HTML which works by updating the DOM. The DOM or Document Object Model will track every change in state that you declare and update the DOM. For example, make the text blue or set the font size to 15

In theory these changes are fast, however, the catch is that frequently manipulating this DOM will make it slow simply because the elements that need to be changed, must first be re-rendered to update what it looks like on your browser end. The solution is to make use of a Virtual DOM, which is a virtual representation of the real DOM ->

How is this better?

For each change that is introduced, a new Virtual DOM will be created to calculate the best and most efficient method to make these changes on the RealDOM. It’s like instead of me telling you all the answers to a math exam, and making you write down some numbers, I’ve told you the most efficient way to get to the answers.

React knows what a component is and how to make them work together whilst ReactDom knows how to make it show up in the DOM. ReactDOM uses javascript to use that collection of components and render it in the DOM. HTML stuff in the render is called JSX, a special dialect which allows us to place javascript code and determine the content of a react app just like normal html.

Screen change on mouse move is called an event handler. They detect when a user is interacting with an app, and update the application/component.

To create a component:

const App = () => {
  return [JSX here]
};

To render the component:

ReactDOM.render(
  <App />,
  document.querySelector('#root')
);

2.2 JSX vs HTML

Adding styling to an individual element

{% raw %} 
HTML: <div style="background-color: red;"><div>
JSX: <div style={{ backgroundColor: 'red' }}></div>

HTML: <div style="background-color: blue; color: white;"><div>
JSX: <div style={{ backgroundColor: 'blue'; color: 'white' }}></div>

First curly brace indicates a javascript class, and the second means a javascript object. You're not supposed to use class in JSX, use className instead to avoid collisions with the keyword “class” which may think we are trying to define a new javascript class

Use a javascript variable and print it inside a JSX block:

const buttonText = 'Click me!';
<button style={{backgroundColor: 'blue', color: 'white'}}>
  {buttonText}
</button>
OR 
function getButtonText() {
  return 'Click on me!';
}
<button style={{backgroundColor: 'blue', color: 'white'}}>
  {getButtonText()}
</button>

There are values JSX can't show. It can show arrays, numbers and strings but not Javascript objects inside of a react Component as text. Unless we use it as a property of the const.
