3. Data Types

3.1 Integers

In C# the variables you need to have an identifier. For example, to define a string in python you would write Number = 5 However, in C# our integer needs to have the int keyword (which defines positive and negative numbers). Hence, to define the same statement in C# it is int Number = 5 Run the following code in the mainline of your HelloWord file:

int a = 18;
int b = 6;
int c = a + b;
Console.WriteLine(c);

When you run it, it should output 24. Other common mathematical operations for integers include:

  • - for subtraction
  • * for multiplication
  • / for division

Question: try multiplying and substring the a and b, then add a third variable d, and add all of the variables.

You may have noticed that integer operations always produce an integer, even when you’d expect the result to print out a decimal. It is important to note that integers round down. For example, try running this code: 

int a = 7;
int b = 4;
int c = 3;
int d = (a + b) / c;
Console.WriteLine(d);

As we can see the expected output of 3.67 (2 decimal places) rounded down to 3. The integer value has an upper and lower value, to run these we can call the MaxValue and MinValue attribute of the integer.

int max = int.MaxValue;
int min = int.MinValue;
Console.WriteLine($"The range of integers is {min} to {max}");

New concept: you may have noticed that we have used the dollar sign ($). It is used in a formatted string (like f strings in python), and allows us to include the variables in our string with the curly braces. It outputs:

The range of integers is -2147483648 to 2147483647
using System;

namespace HelloWorld
{
  class Program
  {
    static void Main(string[], args)
    {
      Console.WriteLine("Hello World!");
      int a = 7;
      int b = 4;
      int c = 3;
      int d = (a + b) / c;
      Console.WriteLine(d);
    }
  }
}

This outputs:

Hello World!
3

In order to get the remainder of our integer division, we can use the remainder operator %. For example, 9 divided by 4 equals 2 with a remainder of 1. In C# performing the operation 9 / 4 would output 2 and 9 % 4 would output one 1.

As we can see the range of the integer variable is very large, we can use. Here is a table that summarises the range of different data types. If our calculation produces a value that exceeds these ranges, we get an underflow or overflow condition. Think of a number line of all integers between -200 and 200. If our calculation equals 202, C# will output -198. It has essentially wrapped our result. If the result is above the maximum range, it is know as an overflow. Meanwhile, if the result is below the negative limit, it is called an underflow error.

Question: in our above example where we have defined an integer to be between -200 and 200 for simplicity, what will the code output if our calculation equals -208?

Answer: It will output 208, known as an underflow.

To test this out, add the following lines to your code.

int what = max + 30;
Console.WriteLine($"An example of overflow: {what}");

3.2 Doubles

Another common data type that you will need is the double – which represents double-precision floating points (15 – 16 digits after the decimal place). Let’s try our same example using doubles and see if it outputs 3.67 (two decimal places).

using System;

namespace HelloWorld
{
  class Program
  {
    static void Main(string[], args)
    {
      Console.WriteLine("Hello World!");
      double a = 7;
      double b = 4;
      double c = 3;
      double d = (a + b) / c;
      Console.WriteLine(d);
    }
  }
}

And this outputs:

3.6666666666666665

3.3 Challenge

Write code to calculate the area of a circle whose radius is 2.50 centimetres. The area of a circle is the radius squared multiplied by PI. One hint: .NET contains a constant for PI, Math.PI that you can use for that value. Math.PI, like all constants declared in the System.Math namespace, is a double value.

3.4 Solution

using System;

namespace HelloWorld
{
  class Program
  {
    static void Main(string[], args)
    {
      double radius = 2.5;
      double area = Math.PI * radius;
      System.Console.WriteLine(area);
    }
  }
}

which gives you

19.634954084936208
