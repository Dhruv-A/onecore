4. Class-Based Components

4.1 Functional vs Class-Based Components

  • Functional components render simple things whilst class-based components introduce more technical elements like LifeCycle Methods and State [We’ll take a look at these in future tutorials]
  • As daunting as it may seem, making a class-based component isn’t too different from a functional component

4.2 Extending a component

Extending a component it allows us to pass props (which stands for properties) to any defined class, without the help of a constructor. Similarly, we can pass around state changes through inheritance by any other class component. However, by extending the react component we must define the ‘render’ method to return the same React element as we did with the functional component

4.3 Rules of a class-based component

  • Must use javascript class
  • Must extend (subclass) React.component
  • Must define a render method that returns JSX

4.4 When to use a class-based component

Use a functional component when you are trying to render simple content. Class components are good for everything else e.g responding to user input, network requests. Whenever you need several helper functions use a class-based component

Benefits of using functional components

  • Easier to understand, read and test because they don’t involve states or lifecycle methods
  • You generally end up typing less code

Benefits of using class-based components

  • Lifecycle methods which occur at different stages when a React component are rendered to the DOM, introduces a slight complexity but makes things easier to do when the App starts up
  • Utilising state enables us to undergo update processes and pass down information to different class-based components. Handles user input
  • Easier code organisation
