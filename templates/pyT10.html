10. Dictionaries

10.1 What is a dictionary?

Dictionaries are unordered collections of data values. Unlike elements of a list, which are indexed by their position, those in a dictionary are indexed by a key. The data in a dictionary is said to be a key:value pair.

Let's say we wanted to make an ID system, which stores information about people based on a unique ID number. We start by making a dictionary which { } and adding keys and values separated with colons

ids = {
  239843: 'Student_A',
  209435: 'Student_B',
  948594: 'Student_C',
  198321: 'Student_D'
}
Download example

This gives us a dictionary where, if we search for someone's ID, it will give us their name. After you download and run the file, try indexing like you did with lists, however, this time use the keys as your index

>>> print(ids[209435])
Student_B
>>> print(ids[0])
Traceback (most recent call last):
  File "<pyshell#3>", line 1, in <module>
    print(ids[0])
KeyError: 0

As you can see, if we try to index using the key, it returns the value stored in the dictionary for that key, however, if we try to index based on position like with lists, it will give us an error. This is because dictionaries will only give you the value corresponding to a key and they are unordered so you can't search for a value using position like you can with lists.

10.2 Iterating through keys

Sometimes, we want to loop through a dictionary, such as if we want to print out all the names in the dictionary in the above example. We can do this using a for loop.

for name in ids:
print(ids[name])

Student_A
Student_B
Student_C
Student_D

Every time you loop through the for loop, the variable name gets assigned to another key in the dictionary, and you simply find the student using that key. This is a very useful way to access the contents of dictionaries.

10.3 Getting values using the get method

When we try to find the value stored in a key in python, we need to make sure that the key exists. In the ids example above, if we try to find the value in ids[3243243] which doesn't exist, we will get an error. To avoid this, python has a get method for dictionaries. It takes two arguments, the first being the key you're trying, and the second being a default value.

>>> print(ids.get(239843, 'Student Not Found'))  # a student who exists in the dictionary
Student_A
>>> print(ids.get(203942, 'Student Not Found'))  # a student who isn't in the dictionary
Student Not Found

As seen, this prevents any errors and lets us assign a default value when a key doesn't exist.

Dictionaries are very useful and important to understand when learning python. They are applicable to solving many different types of problems so it's a good idea to make sure you know how to use them!
